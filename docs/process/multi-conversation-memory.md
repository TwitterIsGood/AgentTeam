# Multi-Conversation Memory

## 分层原则

- **工作记忆**：当前对话中的临时执行上下文
- **长期记忆**：未来仍有价值的规则、事实、关键决策
- **WorkItem checkpoint**：可恢复的当前执行状态

## 长期记忆规则

长期记忆只保留：

- 项目目标
- 架构约束
- 关键决策
- 协作规则
- 持续有效的外部引用

不保留：

- 短期执行细节
- 一次性调试输出
- 可由代码直接推导的事实

## 会话结束时必须生成的 resume block

- WorkItem ID
- 当前阶段
- 已产出工件
- 阻塞点
- 下一步建议

这些信息应写入 `summary.md` 与 checkpoint 文件，而不是只留在聊天记录中。
